## Critical Gap: KPI_CATEGORIES Mismatch (ADMIN-07)

**What:** `AdminKpi.jsx` defines `KPI_CATEGORIES` with long-form values:
```
'Sales & Business Development', 'Operations', 'Finance', 'Marketing', 'Client Satisfaction', 'Team & Culture', 'Custom'
```

Phase 5 migration `006_schema_v11.sql` changed the DB CHECK constraint to short-form:
```sql
CHECK (category IN ('sales', 'ops', 'client', 'team', 'finance'))
```

**Impact:** Any attempt by Trace to create or edit a KPI template sends a category string the database rejects. Template CRUD is completely broken.

**Fix:** Replace `KPI_CATEGORIES` with `['sales', 'ops', 'client', 'team', 'finance']` and use a display labels map for the dropdown UI.

## Verify Intent Items

These are not bugs but design questions surfaced during integration check:

1. **Template count:** SCHEMA-02 says "22 KPI templates" but migration seeds 20 KPI + 8 growth = 28 total. The 22 may refer to KPI+growth combined minus some. Confirm accuracy.

2. **Selection unlock flow:** Pre-seeded mandatory selections are locked. Partners must be unlocked by admin before choosing their 2 additional KPIs. If this is intentional, document it. If not, remove `locked_until` from seed data.

3. **Business growth selection:** SELECT-04 expects partners to "select 2 from 6 options." Implementation shows them as read-only (admin-assigned). If admin-assigned is intentional, update requirement wording.
