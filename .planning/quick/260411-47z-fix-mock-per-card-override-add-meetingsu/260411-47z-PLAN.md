<bug_analysis>
AdminMeetingSessionMock.jsx line 593 (inside KpiStop's PARTNERS.map):
```js
const override = data[p].scorecard?.admin_override_at;
```
This reads a single scorecard-level timestamp. `handleOverrideResult` (line 251) stamps `admin_override_at` on the scorecard whenever ANY kpi is flipped, so the marker bleeds across all 5 KPI cells on the current KPI stop (and across both partner cells on that stop).

Fix: store `override_at` INSIDE each `kpi_results[kpiId]` entry. `handleOverrideResult` already owns the write; change it to stamp `override_at` on the entry, not the scorecard. Then the read on ~593 becomes `entry.override_at`.

The mock builds its own initial data via `buildInitialData()` / `MOCK_SCORECARDS`, so no other file is affected. This also matches the real-DB shape more honestly because future production fixes could pivot the same way (out of scope here).
</bug_analysis>
