## Interaction Contract

### KPI Card Selection (D-03)

- Tap unselected card when count < 5: card transitions to selected state with `selectPulse` animation (existing keyframe). Counter increments.
- Tap unselected card when count = 5: nothing happens. No animation. Counter stays at "5 of 5". Unselected cards receive `.kpi-card.capped` class (opacity 0.45).
- Tap selected card at any count: card deselects (border returns to `var(--border)`, background clears). Counter decrements. If count drops below 5, `.capped` removed from remaining cards.
- "Continue" button: disabled (`.btn-primary:disabled`, opacity 0.35, `cursor: not-allowed`) when `selected.length !== 5`. Enabled only at exactly 5.

### Growth Priority Selection (D-08, D-10)

- Each slot shows predefined options as a list of `.growth-priority-option` buttons.
- Selecting a predefined option deselects any previously selected option in that slot, and hides/dismisses the custom textarea if it was open.
- "Write my own" toggle: appears as the last item in each slot's option list. Clicking it opens a `<textarea>` (existing global styles) and clears any predefined selection. The textarea placeholder text is declared in Copywriting Contract below.
- Custom and predefined are mutually exclusive per slot.

### Confirmation Screen Transition (D-05, D-06)

- "Continue" button on selection screen: validates `selected.length === 5 && personal && business.length === 2`, then persists to Supabase and transitions to confirmation screen. Transition uses Framer Motion `AnimatePresence` with `mode="wait"` â€” same `motion.div` pattern as questionnaire screens (`duration: 0.28, ease: 'easeOut'`).
- Back button on confirmation screen: navigates back to selection screen. State is preserved (selections pre-populated from Supabase on re-mount).
- "Lock In" button: triggers lock write operation, shows `.kpi-lock-success` state, then auto-redirects to `/hub/:partner` after 1800ms.

### Hub Card Three States (D-11)

- **Not started** (no Supabase rows): card renders with CTA "Select Your KPIs", icon `đźŽŻ`, navigates to `/kpi/:partner` on click.
- **In progress** (rows exist, `locked_until` is null): card renders with small gold "In Progress" indicator, CTA "Continue Selection", navigates to `/kpi/:partner`.
- **Locked** (`locked_until` is set): card renders with lock icon `đź”’` replacing the card icon, CTA "View Selections", card `onClick` navigates directly to `/kpi-view/:partner` (not `/kpi/:partner`, per pitfall 5 in RESEARCH.md).

All three states use the same `.hub-card` base class. No new hub-card variant class needed â€” state is expressed via conditional JSX content inside the card, not CSS class swaps.
