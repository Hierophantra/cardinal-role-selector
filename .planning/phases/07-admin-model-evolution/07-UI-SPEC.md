## Component Inventory

Components modified in Phase 7 (no new page-level components):

### AdminKpi ‚Äî KpiTemplateLibrary section

**Mandatory/choice badge row on each template card:**
- Reuse `.kpi-category-tag` visual style for `partner_scope` tag (gold pill: `Shared` / `Theo` / `Jerry`)
- New class `.kpi-scope-tag` ‚Äî identical to `.kpi-category-tag` (gold background tint, gold border, 12px bold uppercase) ‚Äî confirms scope at a glance
- New class `.kpi-mandatory-badge` ‚Äî identical visual to `.kpi-core-badge` (gold tint, 12px bold uppercase, `letter-spacing: 0.12em`) ‚Äî text: `Mandatory` or `Choice`
- Both badges sit on the same horizontal tag row beneath the template label, gap 8px

**Delete button suppression:**
- On mandatory templates: delete button is not rendered (D-01: hidden, not disabled)
- Replace delete button space with inline note: `"Mandatory templates cannot be deleted"` ‚Äî 12px, `var(--muted-2)`, italic

**Edit form ‚Äî measure field addition:**
- Add `<textarea>` for `measure` field after `description` field (D-04)
- Label: `"MEASURE"` ‚Äî follows existing form label pattern (12px uppercase, `var(--muted)`, `letter-spacing: 0.12em`)
- Placeholder: `"How this KPI is tracked (e.g. Weekly pipeline report, CRM updated)"`
- Same `.vision-block` / textarea styling as existing form fields

### AdminPartners ‚Äî Accountability Card (per partner)

**Primary visual anchor:** PIP flag panel (when triggered); ACCOUNTABILITY eyebrow label (when not triggered).

**New card: Accountability** ‚Äî added to each `PartnerSection` below existing data sections.

Structure:
```
[Card container: .admin-card or .scorecard-commit-gate style]
  [Eyebrow: ACCOUNTABILITY]
  [Miss count line: "X missed KPIs across Y submitted weeks"]
  [PIP flag ‚Äî conditional, only when miss count >= 5]
```

**Miss count display (normal state ‚Äî fewer than 5 misses):**
- Container: `var(--surface)` background, `var(--border)` border, `border-radius: 14px`, `padding: 20px 24px`
- Eyebrow: `.eyebrow` class (`var(--red)`, 12px, uppercase) ‚Äî text: `ACCOUNTABILITY`
- Miss count line: 15px, `var(--text)`, weight 400 ‚Äî `"X missed KPIs across Y submitted weeks"`
- Sub-note: 12px, `var(--muted)` ‚Äî `"Only explicit 'No' answers are counted"`
- Zero-miss state: miss count line text uses `var(--success)` color ‚Äî `"No missed KPIs this season"`

**PIP flag (triggered at miss count >= 5):**
- Container: `background: linear-gradient(180deg, rgba(196,30,58,0.12), rgba(196,30,58,0.04))`, `border: 1px solid rgba(196,30,58,0.40)`, `border-left: 3px solid var(--red)`, `border-radius: 10px`, `padding: 16px 20px`, `margin-top: 12px`
- Flag heading: `"‚öÝ Performance Improvement Plan threshold reached"` ‚Äî 15px, `#f2c7cf`, weight 700
- Flag body: `"X missed KPIs counted this season. Review with partner."` ‚Äî 12px, `#f2c7cf`, weight 400

**Card visibility:** Always visible (not collapsible) ‚Äî D-discretion resolved as always-visible; simpler and faster during meetings when Trace needs immediate access.

## Interaction Contracts

### Template Library ‚Äî mandatory badge interactions
- Mandatory templates: no hover transform (cursor: `default` on delete-suppressed area)
- Choice templates: existing edit/delete hover behavior unchanged
- Edit form opens inline (existing pattern in AdminKpi) ‚Äî same for mandatory and choice templates (D-03)

### Label snapshot cascade ‚Äî save button
- When Trace saves a template edit, the save button shows `"Saving..."` text during async operation (existing `saving` state pattern)
- On success: flash message ‚Äî `"Template updated"` (existing `flash` state pattern)
- On cascade failure: error message ‚Äî `"Template saved, but KPI selection labels could not be updated. Refresh and try again."` ‚Äî 12px, `var(--red)`

### Accountability card ‚Äî loading state
- While scorecards are fetching: show `"Loading accountability data..."` in muted text (12px, `var(--muted-2)`)
- Error state: `"Could not load scorecard data."` with a retry link

### PIP flag ‚Äî no interaction
- The PIP flag panel is display-only. No click, no dismiss, no confirmation.
- It remains visible until the cumulative miss count drops below 5 (admin-resets a scorecard via existing reset tools).
