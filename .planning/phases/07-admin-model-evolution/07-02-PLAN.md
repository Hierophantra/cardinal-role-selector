```javascript
const missCount = scorecards.reduce((total, card) => {
  const results = card.kpi_results ?? {};
  return total + Object.values(results).filter((entry) => entry?.result === 'no').length;
}, 0);
const submittedWeekCount = scorecards.filter((s) => s.committed_at).length;
const pipTriggered = missCount >= 5;
```

CRITICAL: Use strict equality `=== 'no'` (D-08). Do NOT use falsy check. `null` and `undefined` results must NOT count as misses.

**Accountability card JSX** — Insert a new section AFTER the "View Scorecard History" link div and BEFORE the "Reset Controls" div. The card renders regardless of loading/lock state (D-06 says it's always visible):

```jsx

<div className="admin-accountability-card">

    </div>
  )}
</div>
```

The accountability card is INSIDE the `<>` fragment that renders when `!loading && !error` — it shares the same conditional rendering gate as the rest of the partner data. Since the loading state is already handled by the parent conditional, no separate loading state is needed for the accountability card itself.
