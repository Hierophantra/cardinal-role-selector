## Spacing Scale

Declared values (must be multiples of 4). Derived from existing patterns in `src/index.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Badge label padding top/bottom, inline icon gaps |
| sm | 8px | Button icon gap, option card gap between items |
| md | 16px | Standard element spacing, card internal padding row gap |
| lg | 24px | KPI cell padding, grid column gap, card section gap |
| xl | 32px | Hero panel padding, meeting stop section gap |
| 2xl | 48px | Meeting stop top/bottom padding |
| 3xl | 64px | Meeting nav bar height, page-level bottom clearance |

Exceptions:
- Week picker label + select + button row uses `gap: 16px` with `marginLeft: auto` on the button — maintain this layout for the dual-CTA row
- Sticky meeting nav bar: fixed `height: 64px`, `padding: 0 40px` — do not alter
- Disabled button touch target: maintain minimum `height: 48px` for the two new CTA buttons

Font weight scale in this phase: **400 (regular)** and **700 (bold)** only.

## Component Inventory

New and modified elements for this phase.

### AdminMeeting.jsx — Landing Page

**Header (no change):** Keep existing `.screen-header` with neutral "Meeting Mode" heading. Source: D-03.

**Dual CTA buttons (new):** Two side-by-side buttons replacing the single "Start Friday Review" button.

```
[ Week picker select — unchanged ]

[ Start Friday Review ]   [ Start Monday Prep ]
```

- Layout: `display: flex; gap: 16px; flex-wrap: wrap; align-items: center`
- "Start Friday Review": `btn btn-primary` — uses `--red` (existing style, no change)
- "Start Monday Prep": `btn btn-primary--monday` — uses `--blue` background

Disabled state (D-02):
- Button is native `disabled` attribute when that meeting type already exists for the selected week
- `opacity: 0.35; cursor: not-allowed` — matches existing `.btn-primary:disabled` rule
- `title` attribute set to `"Already started for this week"` — browser native tooltip, no custom component

**Past meetings list (modified):** Each meeting card gains a type badge (D-10).

Badge anatomy:
- Container: `display: inline-block; padding: 2px 10px; border-radius: 12px; font-size: 12px; font-weight: 700; text-transform: uppercase; letter-spacing: 0.1em`
- Friday Review badge: `background: rgba(196,30,58,0.15); color: var(--red); border: 1px solid rgba(196,30,58,0.3)`
- Monday Prep badge: `background: rgba(37,99,235,0.15); color: var(--blue); border: 1px solid rgba(37,99,235,0.3)`
- No filter UI (D-11) — badges are purely visual

### AdminMeetingSession.jsx — Active and Ended Sessions

**Copy swap (D-04):** `meeting.meeting_type === 'monday_prep'` selects `MONDAY_PREP_COPY`; otherwise `MEETING_COPY`. Ternary at the top of the component, assigned to a local `copy` variable used throughout.

**Monday Prep accent (D-05):** Scoped via a CSS class on the meeting shell root element.

- Class applied to `.meeting-shell` container: add `meeting-shell--monday` when `meeting_type === 'monday_prep'`
- CSS rule targets `.meeting-shell--monday .eyebrow { color: var(--blue); }`
- CSS rule targets `.meeting-shell--monday .meeting-progress-pill { color: var(--blue); border-color: rgba(37,99,235,0.4); }`
- CSS rule targets `.meeting-shell--monday .meeting-kpi-cell.active-highlight { border-color: var(--blue); }` (if active stop highlight is implemented)
- No inline color overrides in JSX — class-based only

**Eyebrow content (D-06):** Eyebrow displays `"FRIDAY REVIEW"` or `"MONDAY PREP"` from the copy constant. No additional badge needed.

**Read-only ended meetings (D-07, D-08, D-09):**
- `textarea` fields: `readOnly` prop set, no cursor (CSS: `cursor: default; resize: none`)
- "End Meeting" button: hidden entirely (`{!isEnded && <button>End Meeting</button>}`)
- Prev/Next navigation: fully active
- Status label: rendered near session header, below the stop eyebrow
  - Copy: `"Ended [formatted date]"` — e.g. "Ended Apr 9, 2026"
  - Style: `font-size: 13px; color: var(--muted); font-style: italic`
  - Position: below `.meeting-shell-header`, above the stop content area
