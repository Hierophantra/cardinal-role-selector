## Typography

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 15px | 400 | 1.55 | Default paragraph, KPI row text, narrative body. Source: `src/index.css` line 53 (`p { line-height: 1.55 }`) and line 70 (input/default 15px). |
| Label / Metadata | 12px | 700 | 1.4 | Uppercase caps labels, status pills, section-header caps, hints, toggle affordances, and inline error text. Letter-spacing 0.1em‚Äì0.18em for caps usage; no letter-spacing for non-caps hints and toggle affordances. |
| Heading (section) | 20px | 700 | 1.3 | Section headings inside hub ("This Week's KPIs", "Personal Growth", "What You Focus On"). Source: `.hub-card h3 { font-size: 20px }` at line 690. |
| Display (role title) | 28px | 700 | 1.2 | Partner role title rendered in `--red`. Source: existing login h1 uses 28px (line 210); role title is the page anchor so it takes the display slot. |

**12px is intentionally multi-use.** It covers two distinct visual sub-roles that share the same size:
- **Caps variant** ‚Äî `text-transform: uppercase; letter-spacing: 0.1em‚Äì0.18em` ‚Äî used for row labels ("Role-mandatory growth"), status pills, section caps. Weight 700.
- **Non-caps variant** ‚Äî no transform, no letter-spacing ‚Äî used for "Read more"/"Show less" toggle affordance, last-week hint, inline "Change" button, and inline error text. Weight 400 (or 700 only where semantically a label).

Executors should treat both as 12px and differentiate visually via `text-transform` and `letter-spacing` only ‚Äî not by assigning different sizes.

Font feature settings: `'ss01', 'cv11'` (global; do not override in new components).
Letter-spacing: headings use `-0.02em` (global `h1, h2, h3, h4` rule at line 48); do not add letter-spacing to body copy.

Italic: `font-style: italic` used for self-quote block only. No other italic elements in Phase 15.

## Color

All values are existing CSS custom properties from `src/index.css` `:root`. Do not introduce new hex values ‚Äî reference variables.

| Role | Variable | Hex | Usage |
|------|----------|-----|-------|
| Dominant (60%) | `--bg` | `#141414` | Page background (radial gradient base) |
| Secondary (30%) | `--surface` / `--surface-2` | `#1E1E1E` / `#242424` | Cards (`.hub-card`), section containers, inline textarea background |
| Accent (10%) | `--red` | `#C41E3A` | Reserved for: role title text, self-quote `border-left` accent, focus ring on textarea, hub-card icon backgrounds (existing) |
| Amber / attention | `--warning` / `--gold` | `#D4A843` | Reserved for: weekly-choice amber card `border-left`, "Not set" pill, section header caps on amber card |
| Semantic: met | `--success` | `#2D8F5E` | Status dot green (met this week), "Locked" pill color |
| Semantic: missed | `--miss` | `#C41E3A` | Status dot red (not met this week) ‚Äî same as `--red` (they are equal; `--miss` alias used for semantic clarity in KPI status context) |
| Muted text | `--muted` | `#AAAAAA` | Status dot gray (not yet answered), secondary labels, narrative muted subtext |
| Border | `--border` | `#2e2e2e` | Card and section container borders |

Accent (`--red`) reserved for exactly:
1. Role title text (`color: var(--red)`)
2. Self-quote left border accent (`border-left: 3px solid var(--red)`)
3. Textarea focus ring (already global; no extra declaration needed)
4. Hub card icon background tint (existing; `rgba(196, 30, 58, 0.10)`)

Do NOT use `--red` for status dots, pills, or ambient decoration. `--miss` handles missed-KPI red semantics.

Source: CONTEXT.md D-10, D-12, D-18; `src/index.css` `:root` block lines 1‚Äì19.

## Interaction Contract

| Interaction | Component | Behavior | State Change |
|-------------|-----------|----------|--------------|
| Click "Read more" / "Show less" | RoleIdentitySection | Toggles narrative expansion. CSS max-height 0 ‚Üí 800px. Label swaps. | `narrativeExpanded` boolean |
| Click "What You Focus On" header | PartnerHub (section toggle) | Collapses open section. Chevron rotates. CSS max-height 1200px ‚Üí 0. | `focusAreasOpen` boolean |
| Click "Your Day Might Involve" header | PartnerHub (section toggle) | Expands collapsed section. Chevron rotates. CSS max-height 0 ‚Üí 600px. | `dayInLifeOpen` boolean |
| Click "Choose your KPI" / "Change" | ThisWeekKpisSection | React Router `<Link>` navigates to `/weekly-kpi/:partner`. No DB write. | Route change only |
| Type in self-chosen textarea | PersonalGrowthSection | Updates draft string in local state. No DB write on keystroke. | `selfChosenDraft` string |
| Click "Save" (growth entry) | PersonalGrowthSection | Calls `upsertGrowthPriority`. On success: row switches to Locked view. On failure: inline error text below textarea. No confirmation dialog. | `growthPriorities` re-fetched or optimistic update |

**Animation budget:** CSS `transition: max-height 0.22s ease` for collapsibles only. No Framer Motion in Phase 15 hub components (D-07). No fade-in animations on section reveal.

**Transition durations:** 0.22s for collapsible max-height, 0.15s for border-color/box-shadow on focused inputs (existing global rule).

## States + Branches

### Hub Loading State

Hub returns `null` during `loading` (existing pattern, line 63 PartnerHub.jsx). Role identity section renders from static data ‚Äî it is NOT gated on loading. Implementation: render `<RoleIdentitySection>` before the loading check by passing static `ROLE_IDENTITY[partner]` directly (available synchronously from import).

### Status Dot States (per KPI row)

| Condition | Dot color |
|-----------|-----------|
| `kpi_results[id].met === true` in current week's scorecard | Green (`--success`) |
| `kpi_results[id].met === false` in current week's scorecard | Red (`--miss`) |
| No scorecard for current week, OR scorecard exists but row is blank | Gray (`--muted-2`) |

Current week = `week_start_date` equal to `getMondayOf(new Date())`.

### Weekly Choice Card States

| Condition | Card content |
|-----------|--------------|
| No `weeklyKpiSelection` for current week | "Choose your KPI for this week" + CTA link |
| `weeklyKpiSelection` exists and scorecard NOT yet submitted | Selected KPI label + "Change" button |
| `weeklyKpiSelection` exists and scorecard submitted | Selected KPI label only (no "Change" button) ‚Äî Phase 16 logic locks it; Phase 15 may show "Change" in all non-null cases since scorecard lock is Phase 16 logic |

**Note:** Phase 15 does not have access to scorecard submission status in a way that cleanly exposes "submitted this week." Simplification for Phase 15: show "Change" whenever selection exists; Phase 16 will add the lock.

### Personal Growth States
