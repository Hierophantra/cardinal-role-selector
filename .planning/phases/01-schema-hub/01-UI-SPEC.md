## Spacing Scale

Declared values (multiples of 4 only). Derived from measured patterns in `src/index.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline item spacing |
| sm | 8px | Button icon gap, grid gaps, small margin offsets |
| md | 16px | Default element spacing, card internal padding row gaps |
| lg | 24px | Container horizontal padding, section gaps |
| xl | 32px | Login card padding, major section breaks |
| 2xl | 48px | Login card vertical padding |
| 3xl | 64px | Page bottom clearance (container `padding-bottom: 80px` — nearest multiple of 4) |

Exceptions:
- Container bottom padding is 80px (not 64px) — existing pattern, preserve as-is
- App header horizontal padding is 28px — existing pattern, preserve as-is
- Card border-radius values follow existing: 10px (input), 12px (btn), 14px (option/summary), 16px (admin-card), 20px (login-card)
- Hub card border-radius: use 16px (matches `.admin-card` — closest existing card for this use case)

Source: `src/index.css` spacing measurements.

## Color

All values are CSS custom properties already defined in `src/index.css` `:root`.

| Role | Value | CSS Var | Usage |
|------|-------|---------|-------|
| Dominant (60%) | `#141414` | `--bg` | Page background (with radial red gradient overlay) |
| Secondary (30%) | `#1E1E1E` / `#242424` | `--surface` / `--surface-2` | Cards, hub option cards, nav |
| Accent (10%) | `#C41E3A` | `--red` | Reserved elements only (see below) |
| Accent alt | `#D4A843` | `--gold` | Category labels, section headers, expandable summaries |
| Success | `#2D8F5E` | `--success` | "Submitted" status pill on admin hub |
| Destructive | `#C41E3A` | `--red` (same as accent) | Destructive actions only |
| Text | `#FAFAFA` | `--text` | All body copy |
| Muted | `#AAAAAA` | `--muted` | Descriptions, meta, labels |
| Border | `#2E2E2E` / `#3A3A3A` | `--border` / `--border-strong` | Card borders, dividers |

**Accent (`--red`) reserved for:**
- `.btn-primary` background (primary CTA only)
- Selected / active state on hub option cards (border + subtle background tint)
- Input focus ring (`box-shadow: 0 0 0 3px rgba(196,30,58,0.18)`)
- `.eyebrow` text color
- Disabled / coming-soon card left-border accent stripe
- Error messages

**Gold (`--gold`) reserved for:**
- Hub card section group header labels (e.g., "Partners", "Accountability")
- `.option .label` category tags inside cards
- Status badges that are informational (not success/error)

**Disabled state** (for admin hub future tools): `opacity: 0.4`, `cursor: not-allowed` — matches `.option:disabled` and `.btn-primary:disabled` patterns.

Source: `src/index.css` `:root` variables + existing component rules.

## Component Inventory

New components needed for Phase 1. All follow existing conventions from `src/index.css` and `CLAUDE.md`.

### Hub Card (`.hub-card`)
Extends `.admin-card` pattern. Large clickable tile with icon area, title, and description.

```
Background: var(--surface)
Border: 1px solid var(--border)
Border-radius: 16px
Padding: 28px
Gap (flex column): 12px
Hover: border-color var(--border-strong), transform: translateY(-1px), transition 0.18s ease
Active/focus: border-color var(--red), box-shadow 0 0 0 1px var(--red) inset
Disabled: opacity 0.4, cursor not-allowed, no hover transform
```

Icon area: 40×40px block, background `rgba(196,30,58,0.10)`, border-radius 10px, centered symbol or text.

### Hub Card Grid (`.hub-grid`)
Two-column grid on desktop, single column on mobile (matches `.admin-grid` breakpoint at 720px).

```
Display: grid
Grid-template-columns: repeat(2, 1fr)
Gap: 20px
Margin-top: 24px
@media (max-width: 720px): grid-template-columns: 1fr
```

### Hub Section Group (`.hub-section`)
Groups hub cards under a labeled section (for admin hub only).

```
Eyebrow label: .eyebrow class (--red, 11px, uppercase, letter-spacing 0.18em)
Margin-bottom between groups: 32px
```

### Status Summary Block (`.status-summary`)
Admin hub only. Surfaces key states at a glance above the card grid.

```
Background: var(--surface)
Border: 1px solid var(--border)
Border-left: 3px solid var(--gold)
Border-radius: 10px
Padding: 16px 18px
Font-size: 14px
Line-height: 1.6
```

Individual status items inside use colored inline dots or text:
- State "complete" / "submitted": color `--success`
- State "pending" / "not yet": color `--muted-2`
- State "locked": color `--gold`

### Partner Greeting (`.partner-greeting`)
Personalized heading block at top of partner hub.

```
Heading: 28px, weight 700, line-height 1.2 — uses .screen-header h2 directly
Status line: 14px, weight 400, color --muted, margin-top 8px
```

Status text is dynamic — pulled from Supabase data state (see Copywriting Contract below).

## Interaction Contract

### Hub Card — Hover
- Transform: `translateY(-1px)`, transition `all 0.18s ease`
- Border color: `var(--border-strong)` — matches `.option:hover` pattern

### Hub Card — Active/Pressed
- `transform: translateY(0)` — reset on mousedown

### Hub Card — Disabled (admin future tools)
- No hover transform
- `opacity: 0.4`, `cursor: not-allowed`
- Small label tag below description showing disabled copy (e.g., "Available in next update")

### Hub Card — Navigation
- Entire card is clickable (`<button>` or `<Link>`)
- No secondary action inside the card

### Page Entry Animation
- Reuse `.fade-in` CSS animation (existing): `opacity: 0 → 1`, `translateY(6px → 0)`, `0.35s ease`
- Apply to `.hub-grid` container, not individual cards (simpler than card stagger for this phase)

### Admin Hub Status Summary
- Static on load — no polling, no real-time updates
- Data fetched once on mount via `useEffect`, follows existing `.catch(console.error)` pattern
