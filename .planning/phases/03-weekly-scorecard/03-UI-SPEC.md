## Animation Contract

Reuse all existing keyframes. No new keyframes defined.

| Interaction | Animation |
|-------------|-----------|
| Page entry | `fade-in` class (existing `fadeIn` keyframe, 0.35s ease) |
| View swap (pre-commit → editing → success) | `AnimatePresence` + `motionProps` matching `KpiSelection.jsx`: `{ opacity: 0, y: 8 }` → `{ opacity: 1, y: 0 }`, duration 0.28s easeOut |
| Reflection textarea reveal | `fadeIn 0.25s ease both` applied to `.scorecard-reflection` |
| History row expand/collapse | Inline height transition: `max-height 0.22s ease`, overflow hidden |
| KPI row border color change on yes/no tap | CSS `transition: border-color 0.18s ease` |
| "Saved" indicator appear/fade | CSS opacity transition 0.3s ease; component manages show/hide timing (appear at 800ms post-save, hide at 2s) |

Standard motion props (shared constant in `Scorecard.jsx`, matching `KpiSelection.jsx`):
```js
const motionProps = {
  initial: { opacity: 0, y: 8 },
  animate: { opacity: 1, y: 0 },
  exit: { opacity: 0, y: -8 },
  transition: { duration: 0.28, ease: 'easeOut' },
};
```

## Interaction Contracts

### Pre-Commit State (Monday gate — D-09)

- Page load on a new Monday week (no `committed_at` for this week): show pre-commit gate view
- KPI labels rendered as read-only preview list (`.scorecard-kpi-preview`) — no yes/no inputs visible
- Single prominent CTA: "Commit to this week" (`.btn-primary`)
- Tapping CTA: calls `commitScorecardWeek(partner, weekOf)` → stamps `committed_at` → swaps view to editing state via `AnimatePresence`
- While `committing` state is true: CTA shows "Committing…" and is `disabled`
- On error: inline error text below CTA: "Could not commit. Try again."

### Editing State (Mon–Sun progressive — D-10, D-15)

- All 5 KPI rows rendered, each with yes/no buttons + collapsed reflection
- Auto-save on every change: yes/no tap triggers immediate upsert; textarea blur triggers upsert with 600ms debounce
- "Saved" indicator appears 800ms after upsert resolves, fades after 2s
- Returning mid-week: pre-filled from DB — yes/no state restored, reflection text pre-filled, reflection textarea visible for any KPI that already has a result
- Counter text updates on each yes/no tap: "X of 5 checked in" (D-02)
- Submit CTA disabled until all 5 reflections non-empty (D-05)
- Week is read-only when `today > sunday_end_of(week_of)` (D-13, D-28) — derived client-side; inputs get `disabled`, no submit button shown, banner reads "This week closed on [date]."

### Success State (post-submit — D-03)

- View swaps to centered success message (reuse `.kpi-lock-success` pattern)
- Success heading: "Check-in submitted"
- Subtext: "See you next week."
- Auto-redirect to `/hub/:partner` after 1800ms

### History Expand/Collapse (D-22)

- Each history row collapsed by default — shows only `.scorecard-history-summary`
- Tap anywhere on the row toggles expanded state
- Expanded: `.scorecard-history-detail` slides in (max-height transition)
- Only one row expands at a time — opening a new row collapses the previously open one
- Expand affordance: a small chevron or "›" character on the right, rotates 90° when expanded (CSS `transform: rotate(90deg)`, transition 0.2s)

### Route Guard

- `/scorecard/:partner` redirects to `/hub/:partner` if KPIs are not locked (D-18)
- `/scorecard/:partner` redirects to `/` if partner slug is invalid (matches existing guard pattern)

## States Required Per Screen

### `/scorecard/:partner` — `Scorecard.jsx`

| State | Trigger | UI |
|-------|---------|-----|
| Loading | Mount, data fetch in-flight | `return null` (matches existing pattern) |
| Load error | fetch throws | Full-screen error message with `SCORECARD_COPY.errorLoad` |
| Pre-commit | KPIs locked, no `committed_at` for current week | Pre-commit gate view |
| Editing | `committed_at` stamped, week is open | Editable scorecard with auto-save |
| Week closed | `today > sunday_end_of(week_of)` | Editable inputs disabled, closed banner |
| Submit success | Submit resolves | Success view → auto-redirect |
| Submit error | Submit throws | Inline error below CTA, inputs remain enabled |

### Hub Scorecard Card — `PartnerHub.jsx`

| State | Condition | CTA Label |
|-------|-----------|-----------|
| Hidden | `!kpiLocked` | Card not rendered |
| Not committed | `kpiLocked && !committedThisWeek` | "Commit to this week →" |
| In progress | `committedThisWeek && answeredCount < 5` | "X of 5 checked in →" |
| Complete | `committedThisWeek && answeredCount === 5 && allReflectionsFilled` | "This week complete ✓" |
