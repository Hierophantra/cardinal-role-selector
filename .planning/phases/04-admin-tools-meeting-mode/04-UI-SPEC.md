## Spacing Scale

Declared values (multiples of 4 only). These match the existing system extracted from `src/index.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, inline badge padding vertical |
| sm | 8px | Gap between status dots, button gaps, inline label spacing |
| md | 16px | Card internal padding (compact), textarea padding, input padding |
| lg | 24px | Section padding, container padding, card gaps |
| xl | 32px | Hub section margin-bottom, wizard stop vertical padding |
| 2xl | 48px | Major section breaks, success screen vertical padding, meeting wizard stop top/bottom padding |
| 3xl | 64px | Page-level spacing (bottom safe area) |

Exceptions:
- Touch targets for Yes/No scorecard override buttons inside Meeting Mode: minimum `48px` height (admin drives fast during live meeting)
- Inline edit inputs inside KPI template cards reuse the existing `.input` CSS rule — no new spacing value declared here.

Note: Meeting wizard stop top/bottom padding uses the 2xl (48px) token — the existing "major section breaks" semantic is appropriate for full-screen wizard stops.

## Color

All tokens are pre-existing CSS custom properties from `src/index.css` `:root`. No new color tokens are introduced in Phase 4.

| Role | Value | CSS Token | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | `#141414` | `var(--bg)` | Page background (via existing radial gradient on `html`) |
| Secondary (30%) | `#1E1E1E` / `#242424` | `var(--surface)` / `var(--surface-2)` | Cards (template editor cards, meeting stop cells, oversight table rows, per-partner sections on AdminPartners) |
| Accent (10%) | `#C41E3A` | `var(--red)` | See reserved list below |
| Destructive | `#C41E3A` | `var(--red)` / `var(--miss)` | Delete template (arm/confirm), unlock KPI (arm/confirm), reopen closed week (arm/confirm) — two-click only |
| Success indicator | `#2D8F5E` | `var(--success)` | Yes result in meeting stop side-by-side view, scorecard dot yes state, hit rate percentage when 5/5 |
| Miss indicator | `#C41E3A` | `var(--miss)` | No result in meeting stop, scorecard dot no state |
| Warning / narrative | `#D4A843` | `var(--gold)` | Status badges (active/achieved/stalled/deferred), category tags, meeting progress pill, admin override marker ("touched by admin") |
| Muted text | `#AAAAAA` / `#777777` | `var(--muted)` / `var(--muted-2)` | Secondary copy, placeholder text, null scorecard dots, disabled state text |

**Accent (`var(--red)`) is reserved for:**
- Focused input/textarea border + box-shadow glow
- Selected KPI card border + gradient fill (template editor selected state)
- Meeting Mode hero card left-border accent stripe on AdminHub
- Active/in-progress meeting indicator badge on AdminHub status summary
- Primary CTA buttons (`btn-primary` class): "Start Meeting", "Lock Changes", "Save Template", "Confirm Unlock KPIs"
- Destructive armed-state buttons: second click in two-click arm/confirm sequence
- Eyebrow labels (`var(--red)`) — unchanged from existing pattern

**Gold (`var(--gold)`) is reserved for:**
- Growth priority status badge (all four states use gold background tint; state label text differentiates them — see Copywriting Contract)
- KPI category tags (`.kpi-category-tag` — unchanged)
- Meeting progress pill text (stop X of 10)
- Admin override / "touched by admin" muted marker on scorecard history rows
- Auto-save confirmation flash ("Saved" text in meeting notes)

## Interaction Contract

### Meeting Mode Wizard

- Entry: Admin clicks "Start Meeting" on `/admin/meeting` landing page. Creates `meetings` row immediately. Navigates to `/admin/meeting/:id` at stop `intro`.
- Navigation: Prev/Next buttons in `.meeting-nav`. No keyboard shortcut in v1. Progress pill is display-only (not clickable — agenda order is fixed).
- Per-stop notes: textarea auto-saves via debounced upsert (400ms debounce) to `meeting_notes` on `(meeting_id, agenda_stop_key)`. "Saved" flash in gold fades in/out on successful write.
- Scorecard override: Yes/No override in `.meeting-yn-override` calls `adminOverrideScorecardEntry` immediately on click. Stamps `admin_override_at`. No confirmation required (low-stakes, undoable by clicking the other button).
- End Meeting: "End Meeting" button in header is two-click arm/confirm. First click arms. Second click stamps `ended_at` and navigates back to `/admin/meeting` landing.
- Transitions: Framer Motion `AnimatePresence` with `mode="wait"`. Stop transition: `initial={{ opacity: 0, x: 24 }}`, `animate={{ opacity: 1, x: 0 }}`, `exit={{ opacity: 0, x: -24 }}`, `transition={{ duration: 0.22, ease: 'easeOut' }}`. (Matches existing `{ duration: 0.28, ease: 'easeOut' }` pattern, slightly faster for meeting pace.)

### KPI Template Editor

- Cards render in `.kpi-list` (existing class) with inline "Edit Template" button visible on hover (`opacity: 0` default, `opacity: 1` on `.kpi-template-editor-card:hover`).
- Clicking "Edit Template" swaps label/description/category to inputs in-place. Card transitions to `.kpi-template-editor-card.editing` state.
- "Save Template" submits. "Discard Changes" discards. No modal. Both buttons appear in `.kpi-template-editor-actions`.
- "Delete Template" button: two-click arm/confirm. First click shows "Confirm Delete?" text with a red-accented confirm button and a "Keep Template" ghost button. Three-second auto-disarm (cancel armed state after 3s of inactivity — matching existing AdminPartners ResetButton auto-disarm convention).
- "+ Add Template" card at bottom of list opens a blank editing card in place.
- Category field: `<select>` element with options sales / ops / client / team. Styled to match existing `input` rules.

### Growth Priority Status Toggle (AdminPartners / /admin/kpi)

- Status displayed as `.growth-status-badge` with partner's current status.
- Admin interaction: click-to-cycle through `active → achieved → stalled → deferred → active`. No confirmation. Persists immediately via `updateGrowthPriorityStatus`. Gold flash "Saved" on success.
- Admin note: textarea below badge. Saves on blur via `updateGrowthPriorityAdminNote`. "Saved" flash on success.

### Scorecard Reopen (admin/scorecards only)

- "Reopen Week" button per closed week row. Two-click arm/confirm. Confirm triggers `reopenScorecardWeek`. Badge `.scorecard-reopened-badge` "Reopened" appears inline after reopen.

### Unlock Partner KPIs

- "Unlock KPIs" button on AdminPartners per-partner section and on `/admin/kpi` with partner selector. Two-click arm/confirm. Warning copy: "This lets {Partner} re-select KPIs. Their current picks are preserved. Re-lock starts a new 90-day period."
