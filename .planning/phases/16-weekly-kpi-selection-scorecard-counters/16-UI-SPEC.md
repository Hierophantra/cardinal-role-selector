## Color

All values extend Phase 15 color contract. No new hex values тАФ reference existing CSS custom properties only.

| Role | Variable | Hex | Usage |
|------|----------|-----|-------|
| Dominant (60%) | `--bg` | `#141414` | Page background |
| Secondary (30%) | `--surface` / `--surface-2` | `#1E1E1E` / `#242424` | Selection cards, scorecard rows, modal background |
| Accent (10%) | `--red` | `#C41E3A` | Reserved for: selected-card border + inset shadow on selection grid; focus ring on inputs (global); submit button (`.btn-primary` background) |
| Amber / attention | `--warning` / `--gold` | `#D4A843` | Reserved for: weekly-choice amber card (existing HUB-03); category tag on selection cards; "Used last week" disabled-card label |
| Semantic: met | `--success` | `#2D8F5E` | Scorecard row left-border when Met; "Met" active button background tint |
| Semantic: missed | `--miss` | `#C41E3A` | Scorecard row left-border when Not Met; "Not Met" active button background tint; inline error text |
| Muted text | `--muted` | `#AAAAAA` | Disabled-card opacity overlay; modal sub-copy; counter label |
| Border | `--border` | `#2e2e2e` | Card borders, scorecard row borders, modal border |

Accent (`--red`) reserved for exactly:
1. Selected KPI card border + inset shadow (`.kpi-card.selected` тАФ existing class, reused)
2. Focus ring on all inputs/textareas (existing global rule)
3. Primary submit button background (`.btn-primary` тАФ existing class)
4. Scorecard row left-border when result is Not Met (`.scorecard-kpi-row.no` тАФ existing class)

Do NOT use `--red` for disabled-card labels, modal headings, or counter pills. `--miss` and `--gold` handle their semantic roles.

Source: Phase 15 UI-SPEC color section; CONTEXT.md D-01 (disabled card opacity 0.45); `src/index.css` lines 1тАУ19, 771тАУ781.

## Interaction Contract

| Interaction | Component | Behavior | State Change |
|-------------|-----------|----------|--------------|
| Tap KPI card on selection screen | WeeklyKpiSelectionFlow | Highlights card (`.kpi-card.selected`), advances to confirmation step. Previous-week card is non-interactive. | `selectedKpi` object; `view тЖТ 'confirmation'` |
| Confirm on confirmation step | WeeklyKpiSelectionFlow | Calls `upsertWeeklyKpiSelection`. On success: `view тЖТ 'success'`. On back-to-back error: `view тЖТ 'selection'` + inline error shown. On other error: inline error on confirmation step. | `view`; `committed` boolean |
| "Go back" on confirmation step | WeeklyKpiSelectionFlow | Returns to selection step without DB write. Selected card stays highlighted. | `view тЖТ 'selection'` |
| "Back to Hub" on success step | WeeklyKpiSelectionFlow | React Router navigate to `/hub/:partner` | Route change |
| Tap "+1" on counter pill | ThisWeekKpisSection (inline) | Optimistic local count +1. Debounced 500ms write to `incrementKpiCounter`. | `counterValues` map (local state); Supabase write after 500ms |
| Toggle Met / Not Met | Scorecard | Highlights active button; updates JSONB draft. Row border shifts color. | `kpiResults` draft object |
| Type in count field | Scorecard | Updates `kpiResults[id].count` in draft. No DB write on keystroke. | `kpiResults` draft object |
| Type in reflection | Scorecard | Updates `kpiResults[id].reflection` in draft. No DB write on keystroke. | `kpiResults` draft object |
| Type in weekly reflection fields | Scorecard | Updates `weeklyReflection` draft object. No DB write on keystroke. | `weeklyReflection` draft object |
| Click "Submit Scorecard" | Scorecard | Validates all 7+ KPI rows have a Met/Not Met answer. If incomplete: inline error at sticky bar. If complete: writes to `scorecards` + marks `weekly_kpi_selections.scorecard_submitted = true`. On success: transitions to read-only view. | `submitted` boolean; read-only render |
| Click "Go to Hub" (empty guard) | Scorecard | React Router navigate to `/hub/:partner` | Route change |

**Animation budget:**
- Step transitions in WeeklyKpiSelectionFlow: Framer Motion, `duration: 0.28, ease: 'easeOut'` (matches questionnaire pattern per CONTEXT code_context)
- Scorecard row color transition on Met/Not Met toggle: `transition: border-color 0.18s ease` (existing `.scorecard-kpi-row` rule)
- Counter optimistic increment: instant local update тАФ no animation
- No other animations in Phase 16
