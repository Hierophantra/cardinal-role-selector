# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Partner & Meeting Flow Updates.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla CSS only |
| Preset | not applicable |
| Component library | none — custom Cardinal CSS classes in `src/index.css` |
| Icon library | Unicode emoji / inline SVG (existing pattern) |
| Font | DM Sans (Google Fonts), system-ui fallback |

**Rationale:** CLAUDE.md hard-constrains the stack to "vanilla CSS (must stay consistent with existing)". No component library is in scope. All new CSS classes extend `src/index.css` using the established naming pattern.

**No shadcn gate required.** Project is explicitly vanilla CSS; shadcn initialization would violate the stack constraint.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (horizontal 4px), dot gaps |
| sm | 8px | Gap between KPI card internal elements, button gaps, badge padding (vertical 8px) |
| md | 16px | Section internal padding, input padding, growth field gaps |
| lg | 24px | KPI card padding, confirmation section padding, scorecard section padding |
| xl | 32px | Between major sections (mandatory list ↔ choice section, KPI rows ↔ weekly reflection) |
| 2xl | 48px | Meeting stop top padding, lock-success vertical padding |
| 3xl | 64px | Page-level bottom padding (container default) |

Exceptions:
- Touch targets (yes/no buttons, week-rating buttons): minimum 44px height — source: existing `.scorecard-yn-btn` pattern at `height: 44px`
- Meeting nav bar: fixed 64px height — source: existing `.meeting-nav` at `height: 64px`
- Meeting header: fixed 56px height — source: existing `.meeting-shell-header` at `height: 56px`

## Typography

Source: existing `src/index.css` global declarations. No new font sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / KPI label | 15px | 400 | 1.55 |
| Label / meta / badge text | 12px | 700 | 1.2 |
| Subtext / description / muted | 13–14px | 400 | 1.5–1.6 |
| Heading / screen title | 28px | 700 | 1.2 |

Notes (all sourced from existing CSS):
- Body: `.kpi-card-label`, `.scorecard-kpi-label`, `.meeting-stop-subtext` all use `font-size: 15px; line-height: 1.55`
- Labels: `.kpi-category-tag`, `.growth-priority-group-label`, `.meeting-partner-name` all use `font-size: 12px; font-weight: 700; letter-spacing: 0.12–0.14em; text-transform: uppercase`
- Headings: `.screen-header h2` uses `font-size: 28px; line-height: 1.2`
- No new font sizes. The "Core" badge and week-rating labels fit the 12px/700 label role.

## Color

Source: CSS custom properties in `:root` block of `src/index.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#141414` (`var(--bg)`) | Page background (with red radial gradient overlay) |
| Secondary (30%) | `#1E1E1E` / `#242424` (`var(--surface)` / `var(--surface-2)`) | Cards, locked KPI list, scorecard rows, meeting KPI cells, text inputs |
| Accent (10%) | `#C41E3A` (`var(--red)`) | Selected states, left-border on active rows, primary CTA button, focus rings |
| Core badge accent | `#D4A843` (`var(--gold)`) | "Core" badge only — text, border, and 10% opacity background fill |
| Destructive | `#C41E3A` (`var(--red)`) | Same as accent; this project uses red for both interactive selection and destructive framing |
| Success | `#2D8F5E` (`var(--success)`) | "Yes" answered KPIs, left-border on yes rows, hit-rate indicators |
| Miss | `#C41E3A` (`var(--miss)`) | "No" answered KPIs, left-border on no rows |

**Accent reserved for:**
- Selected KPI card border + background gradient (`kpi-card.selected`)
- "Yes" button active state uses `var(--success)`; "No" button active state uses `var(--miss)`
- Primary CTA button background (`btn-primary`)
- Input focus ring (`box-shadow: 0 0 0 3px rgba(196,30,58,0.18)`)
- Eyebrow text color (`.eyebrow`)
- Lock-in notice left border (`.kpi-locked-notice`)

**Core badge color is gold (`var(--gold)`)**, not red, to distinguish it from interactive selection states. The gold palette is already used for category tags (`.kpi-category-tag`) — the Core badge shares this treatment and is visually consistent.

**Week rating selected state:** Use `rgba(212,168,67,0.14)` background + `var(--gold)` border for selected rating button, matching the `.kpi-counter.at-cap` gold highlight pattern. This keeps rating selection visually distinct from KPI yes/no (which uses red/green).

## Interaction Contracts

### Selection Screen: Mandatory Section

- Mandatory KPI rows render as non-interactive (`cursor: default`, no hover transform, no click handler)
- Each row shows: `.kpi-core-badge` + KPI label + measure text below in muted color
- Gold left border (3px `var(--gold)`) visually locks the row without using `opacity: 0.4` (which implies disabled)
- Choice counter shows "0 / 2 chosen" progressing to "2 / 2 chosen" — reuse `.kpi-counter` / `.kpi-counter.at-cap` pattern
- "Confirm selections" CTA (`.btn-primary`) is disabled until exactly 2 choice KPIs are selected
- Transition to confirmation view: `motionProps` with `{ duration: 0.28, ease: 'easeOut' }` — matches existing pattern

### Selection Screen: Confirmation View

- Lists all 7 KPIs in a single `.kpi-confirmation-screen` section
- Mandatory entries show `.kpi-core-badge`; choice entries show no badge
- Growth priorities section below KPIs:
  - Mandatory personal priority: read-only display row
  - Self-chosen personal priority: read-only display of entered title + measure
  - Business priorities: if assigned by Trace, listed read-only; if not yet assigned, show empty state copy (see Copywriting Contract)
- Single CTA: "Lock in for Spring Season 2026" (`.btn-primary`)
- Lock confirmation uses `.kpi-locked-notice` styling with "Spring Season 2026" language

### Scorecard: 7 KPI Rows

- Mandatory KPIs render `.kpi-core-badge` inline after the KPI label in `.scorecard-kpi-label`
- Yes/No button touch targets: 44px height (existing `.scorecard-yn-btn` pattern)
- Reflection textarea: auto-saves with 400ms debounce (existing `Scorecard.jsx` pattern carries forward)
- Weekly Reflection section gated: only visible after all 7 KPI yes/no values are set (same gate as existing check-in logic)
- Submit button disabled until: all 7 KPIs answered + `weekly_win` filled + `week_rating` selected (1–5)
- `tasks_completed`, `tasks_carried_over`, `weekly_learning` are optional (placeholder text indicates this)

### Meeting Mode: 7 KPI Stops

- Stop header renders: eyebrow text "KPI {N}" + `.kpi-core-badge` if mandatory
- Example rendering: "KPI 1: Revenue Growth [Core]" = eyebrow "KPI 1" + `.meeting-stop-heading` "Revenue Growth" + `.kpi-core-badge`
- STOPS array order: mandatory KPIs first (kpi_1 through kpi_5), then choice KPIs (kpi_6, kpi_7), then growth and wrap-up stops
- No structural change to `meeting-kpi-grid` or `meeting-kpi-cell` — 2-column side-by-side layout continues
- "Core" tag in stop header uses `.kpi-core-badge` (same class, not a meeting-specific variant)

### Framer Motion Transitions

All view transitions use the shared `motionProps` constant:
```js
{ initial: { opacity: 0, y: 8 }, animate: { opacity: 1, y: 0 }, exit: { opacity: 0, y: -8 }, transition: { duration: 0.28, ease: 'easeOut' } }
```
Source: existing pattern in `KpiSelection.jsx` and `Scorecard.jsx`.
