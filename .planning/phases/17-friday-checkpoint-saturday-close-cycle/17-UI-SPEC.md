### Scorecard row picker (3-button extension ‚Äî D-05, D-06, KPI-01, KPI-02)

Applied to **`Scorecard.jsx`** lines 575‚Äì594 (existing 2-button row).

**Button row layout:**

The existing `.scorecard-yn-row` becomes a 3-child flex row. Three buttons side-by-side, equal visual weight:

```
[ Met ]  [ Not Met ]  [ Pending ]
```

- All three buttons reuse `.scorecard-yn-btn` (height 44px, padding 10px 24px, border-radius 10px, font 14px/700)
- Active states:
  - `.scorecard-yn-btn.yes.active` ‚Äî green tint (existing, unchanged)
  - `.scorecard-yn-btn.no.active` ‚Äî red tint (existing, unchanged)
  - `.scorecard-yn-btn.pending.active` ‚Äî **NEW: amber tint** ‚Äî `background: rgba(212,168,67,0.18); border-color: var(--gold); color: var(--text)`

**Pending textarea reveal (D-06):**

When a row's `result === 'pending'`, a textarea reveals **inline below the button row, above the Reflection block**. Same vertical locus as the existing reflection field but rendered separately.

- Container: `.scorecard-pending-reveal` ‚Äî `max-height: 0; overflow: hidden; transition: max-height 0.22s ease`
- Expanded modifier: `.scorecard-pending-reveal.expanded` ‚Äî `max-height: 200px`
- Inner content rendered when result is `'pending'` (so React unmounts the textarea cleanly when toggled away ‚Äî clearing field per D-06)
- Label: `"Follow-through commitment"` ‚Äî 12px, weight 700, uppercase, letter-spacing 0.1em, `color: var(--muted)` (matches existing `.scorecard-reflection-label`)
- Textarea: standard `<textarea>`, `rows={2}`, placeholder copy below
- Reuses existing global textarea styles (`src/index.css:63‚Äì81`) ‚Äî focus ring stays red per global rule

**Toggle-away clearing (D-06):**

When the partner clicks Yes or No while result was `'pending'` and `pending_text` is non-empty:
- Field is **cleared silently** ‚Äî no confirmation modal, no toast
- React state for `pending_text` resets to `''`
- The `.scorecard-pending-reveal` collapses (transitions max-height back to 0) on the next render because `result !== 'pending'`

**Submit gate (KPI-02):**

`handleSubmit` in `Scorecard.jsx:302` extends its incomplete-row check. Current check:
```
const r = kpiResults[tpl.id]?.result;
return r !== 'yes' && r !== 'no';
```
becomes:
```
const r = kpiResults[tpl.id]?.result;
const t = kpiResults[tpl.id]?.pending_text ?? '';
if (r !== 'yes' && r !== 'no' && r !== 'pending') return true;  // not rated
if (r === 'pending' && t.trim() === '') return true;             // pending without commitment
return false;
```
Inline error reuses `SCORECARD_COPY.submitErrorIncomplete` (existing) plus a new `SCORECARD_COPY.submitErrorPendingTextRequired` for the specific pending-no-text case (see Copywriting Contract).

**Row state coloring (extended ‚Äî D-07):**

`.scorecard-kpi-row` gets two new modifiers:
- `.scorecard-kpi-row.pending` ‚Äî `border-left: 3px solid var(--gold); border-color: rgba(212, 168, 67, 0.4)` ‚Äî mirrors existing `.yes` / `.no` patterns at `src/index.css:958‚Äì966`
- `.scorecard-kpi-row.pending.muted` ‚Äî `border-left: 3px solid var(--muted-2); border-color: rgba(119, 119, 119, 0.4)` ‚Äî applied wherever read-only render shows a Pending row after Saturday close (`effectiveResult === 'no'`, raw === `'pending'`)

The class is applied based on the **raw** `result` value plus `isWeekClosed(weekOf)`:
- raw === 'yes' ‚Üí `.yes`
- raw === 'no' ‚Üí `.no`
- raw === 'pending' AND week open ‚Üí `.pending`
- raw === 'pending' AND week closed ‚Üí `.pending.muted`
- raw === null/undefined ‚Üí no modifier

**Pending badge component (D-07):**

A small text badge rendered next to the result wherever a Pending row surfaces (inline beside the result label in read-only Scorecard, hub history rows, comparison view, Friday meeting kpi_* stops).

- Class: `.pending-badge` (live amber state)
- Modifier: `.pending-badge.muted` (post-close `Pending ‚Üí No` state)
- CSS new:
  - `.pending-badge` ‚Äî `display: inline-block; font-size: 12px; font-weight: 700; text-transform: uppercase; letter-spacing: 0.1em; border-radius: 6px; padding: 4px 8px; color: var(--gold); background: rgba(212, 168, 67, 0.10); border: 1px solid rgba(212, 168, 67, 0.30); margin-left: 8px; vertical-align: middle`
  - `.pending-badge.muted` ‚Äî `color: var(--muted-2); background: var(--surface-2); border: 1px solid var(--border)`

**Re-open Pending rows post-submit (D-16):**

When the partner returns to `Scorecard.jsx` after submit AND the week is still open (`!isWeekClosed(currentWeekOf)`) AND at least one row has raw result `'pending'`:
- The view stays in "submitted" read-only mode for Yes/No rows (existing behavior preserved)
- Pending rows specifically render the **3-button picker + textarea** in editable mode
- Visual differentiator: each editable Pending row has a small italic muted note above its picker: `"Update Pending row ‚Äî locks at Saturday 23:59"` (12px, italic, `var(--muted-2)`)
- A second submit button appears in the sticky bar with copy `"Update Pending Rows"` (replaces the original `"Submit Scorecard"` after first submit)
- After Saturday close (`isWeekClosed(currentWeekOf)`) the entire view is fully read-only ‚Äî no pickers render anywhere

**New CSS classes:**
- `.scorecard-yn-btn.pending` ‚Äî base color treatment for the inactive Pending button: `color: var(--muted)` ‚Äî same as `.yes` / `.no` base
- `.scorecard-yn-btn.pending.active` ‚Äî `background: rgba(212, 168, 67, 0.18); border-color: var(--gold); color: var(--text)`
- `.scorecard-kpi-row.pending` ‚Äî `border-color: rgba(212, 168, 67, 0.4); border-left: 3px solid var(--gold)`

### KpiReviewOptionalStop (gate stop ‚Äî MEET-07, D-09, D-10, D-12, D-17)
