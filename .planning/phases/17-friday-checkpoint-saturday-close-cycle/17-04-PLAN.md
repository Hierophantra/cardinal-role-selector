**Edit 3 — Add `SaturdayRecapStop` inline component (D-11, D-15, D-16):**

This is the most complex new component. Per researcher Q1 strategy (a) — Wave 2 preserves `pending_text` on yes-conversion via `buildKpiResultsPayload` retention. Therefore: a row qualifies for the recap if `entry.pending_text` is non-empty (regardless of current `entry.result`). Conversion state derives from `effectiveResult(entry.result, weekOf)` — `'yes'` means the partner converted, anything else means did-not-convert.

```jsx
function SaturdayRecapStop({ meeting, lastWeekScorecards, notes, savedFlash, onNoteChange, copy, isEnded }) {
  // lastWeekScorecards is an array of { partner, kpi_results, week_of, ... } for both partners,
  // for the Monday immediately prior to meeting.week_of. Provided by the parent's data fetch
  // (see Edit 5 — adds `lastWeekScorecards` to the fetch / passes via data prop).

  const recapRows = [];
  for (const sc of lastWeekScorecards ?? []) {
    for (const [tplId, entry] of Object.entries(sc?.kpi_results ?? {})) {
      const pendingText = (entry?.pending_text ?? '').trim();
      if (!pendingText) continue;
      // entry has follow-through commitment text — was Pending at some point last week.
      const eff = effectiveResult(entry.result, sc.week_of);
      const converted = eff === 'yes';
      recapRows.push({
        partner: sc.partner,
        tplId,
        label: entry.label ?? '(KPI)',
        pending_text: entry.pending_text,
        converted,
      });
    }
  }

  const empty = recapRows.length === 0;

  return (
    <>
      <div className="eyebrow meeting-stop-eyebrow">{copy.stops.saturdayRecapEyebrow}</div>
      <h3 className="meeting-stop-heading">{copy.stops.saturdayRecapHeading}</h3>

      {empty ? (
        <div className="saturday-recap-empty">{copy.stops.saturdayRecapEmpty}</div>
      ) : (
        <div className="saturday-recap-list">
          {recapRows.map((row, i) => (
            <div key={`${row.partner}-${row.tplId}-${i}`} className="saturday-recap-row">
              <div className="saturday-recap-label" style={{ fontSize: 15, fontWeight: 700, color: 'var(--text)' }}>
                {PARTNER_DISPLAY[row.partner] ?? row.partner}: {row.label}
              </div>
              <div className="saturday-recap-commitment">
                {copy.stops.saturdayRecapCommitmentPrefix}{row.pending_text}
              </div>
              <div className={`saturday-recap-conversion ${row.converted ? 'met' : 'not-converted'}`}>
                {row.converted ? copy.stops.saturdayRecapMet : copy.stops.saturdayRecapNotConverted}
              </div>
            </div>
          ))}
        </div>
      )}

      <StopNotesArea
        stopKey="saturday_recap"
        notes={notes}
        savedFlash={savedFlash}
        onNoteChange={onNoteChange}
        copy={copy}
        isEnded={isEnded}
      />
    </>
  );
}
```

`PARTNER_DISPLAY` is the existing import from `'../../data/content.js'` (mapping partner slug → display name). If not currently imported in this file, add it to the existing destructured import.
