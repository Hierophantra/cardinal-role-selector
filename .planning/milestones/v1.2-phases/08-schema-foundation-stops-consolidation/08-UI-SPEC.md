## Copywriting Contract

### MONDAY_PREP_COPY — Full Contract

This is the primary copywriting deliverable of Phase 8. The constant is authored here and implemented during execution. It follows the exact shape of `MEETING_COPY` (content.js lines 580–608).

Source for all framing decisions: RESEARCH.md — Monday Prep orientation is forward-looking ("What's the plan for...") vs. Friday Review retrospective ("How did...").

| Key path | Copy | Notes |
|----------|------|-------|
| `landingEyebrow` | `'MEETING MODE'` | Same as MEETING_COPY — landing page eyebrow is type-agnostic |
| `startCta` | `'Start Monday Prep'` | Distinguishes from Friday's "Start Meeting" on the landing card |
| `heroCardTitle` | `'Monday Prep'` | Used in hero card heading if Phase 9 renders it |
| `heroCardDescription` | `'Set targets and commitments before the week begins.'` | Placeholder — user will refine in Phase 9 (Claude's Discretion per CONTEXT.md D-07) |
| `progressPill` | `(n, total) => \`Stop \${n} of \${total}\`` | Same function signature as MEETING_COPY — identical display |
| `weekPickerLabel` | `'Week:'` | Same — week picker UI is type-agnostic |
| `endBtn` | `'End Prep'` | Shorter, distinct from "End Meeting" |
| `endConfirmBtn` | `'Confirm End'` | Same as MEETING_COPY |
| `endedNav` | `'Back to Meeting History'` | Same — history navigation is type-agnostic |
| `landingEmpty` | `'No Monday Prep sessions yet. Start your first prep session.'` | Parallel structure to MEETING_COPY landingEmpty |
| `stops.introEyebrow` | `'MONDAY PREP'` | Replaces 'FRIDAY REVIEW' — uppercase, renders in `.eyebrow` |

## Interaction Contracts

### Defect Fix: kpi_6 and kpi_7 Notes Visible in MeetingSummary

**Before:** MeetingSummary renders 10 stops. If a meeting had notes on kpi_6 or kpi_7, those notes were silently absent from the partner's summary view.

**After:** MeetingSummary renders 12 stops via `AGENDA_STOPS`. Notes for all 7 KPI stops appear in stop order.

**Interaction contract:** No interaction change. The component renders a list of stop-by-stop notes. The list is now complete. Existing stop order and rendering logic is unchanged.

### No New Interactions

Phase 8 adds no new buttons, forms, modals, or navigation. The `MONDAY_PREP_COPY` constant is authored but not connected to any UI until Phase 9.
